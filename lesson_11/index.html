Задається трансформація властивістю transform та її значеннями. При вказані відсоткових значень відсотки беруться від розміру об’єкта x - від ширини y - від висоти

2D Трансформації

у

translate X,Y = translate(10%,10%)

к

translate X = translate(10%,0%)

р

scale XY = scale(50%,50%)

а

scale XY = scale(130%,130%)

ї

scale N = scale(70%)

н

rotate (deg) = rotate(30deg)

а

rotate X (deg) = rotateX(120deg)

в

rotate Y (deg) = rotateY(180deg)

і

transform-origin: 0 0; transform: rotate(360deg);

л

skew (xdeg, ydeg); = skew(-10deg, 0deg)

ь

transform:(translate, scale, rotate, skew);

н

new translate = translate: 0% 10%

а

new scale = scale: 2 1;

!

new rotate = rotate:70deg

важливо:

Нові окремі властивості: translate, scale, rotate не підтримуються валідатором CSS (у валідаторі видає помилку на цю властивість).

При використанні transform об’єкт стає ближче до користувача по осі z, тобто накриває собою інші об’єкти яким не задано властивостей transform та position, але керувати цими об’єктами за допомогою z-index ми не можемо, тому що z-index працює лише з елементами позиція яких відмінна від static.

При застосуванні scale зі значенням більше за 1 для картинок, буде псуватись якість картинки тому щоб уникнути цього варто експортувати картинку трішки більшого розміру та вказувати для неї макетні розміри за допомогою властивості max-width
